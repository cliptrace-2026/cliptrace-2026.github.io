入门指南

Starter Kit、环境配置、基线方法与提交步骤

Starter Kit

CLIPTrace Challenge Starter Kit 将提供参加比赛所需的模型、代码脚本、提交模板和基线实现，帮助参赛者快速复现比赛环境并开始方法开发。

Starter Kit GitHub 仓库即将公开。正式地址将在数据和竞赛平台开放时更新。

模型与环境

竞赛模型采用 Hugging Face Transformers 格式的 CLIP ViT-B/32，对应 224×224 输入、32×32 patch 和 512 维投影特征。参赛者可以读取完整模型权重并访问中间层表示。

from transformers import CLIPModel model = CLIPModel.from_pretrained(MODEL_PATH) model.eval()

Starter Kit 将提供推荐的 Python 环境、依赖版本和模型加载示例。

Starter Kit 内容

  • 模型：CLIP ViT-B/32 开发阶段模型及模型清单。
  • 代码脚本：模型加载、预测生成、embedding 校验和 ZIP 打包脚本。
  • 提交模板：predictions.json 和 .pt 特征文件示例。
  • 基线实现：基于 DECREE 的参考后门检测和分析方法。
  • 校验工具：检查 model_id、JSON 字段、Tensor 形状、dtype 和归一化的脚本。

参赛流程

  1. 注册比赛。
    在 Codabench 页面创建队伍并完成报名。平台链接即将公布。
  2. 下载 Starter Kit。
    获取模型、环境配置、基线代码和提交模板。
  3. 生成预测。
    对每个模型预测 0/1 标签，并为预测为后门的模型生成目标 embedding。
  4. 打包提交。
    将 predictions.json 和 embeddings/ 目录打包为一个 ZIP 文件。
  5. 上传并查看结果。
    在 Codabench 的 My Submissions 页面上传 ZIP 并查看得分。

提交次数

阶段每个账户每日上限阶段总上限
开发阶段3 次不单独限制
测试阶段2 次不单独限制

基线方法

Starter Kit 将提供基于 DECREE 的参考实现，用于展示后门模型分析的基本流程。基线仅用于帮助参赛者熟悉数据和提交接口，不代表最终推荐方案。

完整实现、运行示例和参数说明将在 Starter Kit 仓库中提供。
