规则与常见问题

队伍、公平性、外部资源和结果复现

队伍与公平性

  • 每支队伍最多 5 人，每位参赛者只能加入一支队伍。
  • 禁止通过多账户或多队伍规避提交限制。
  • 禁止利用隐藏标签、目标向量、数据泄漏或平台漏洞获取不公平优势。
  • 组织者、数据生成成员、评分器开发成员及提前接触隐藏答案的人员不得参赛。
  • 其他潜在利益冲突应主动向组织者申报。

外部资源与复现

允许使用公开数据、公开预训练模型、公开生成模型、参赛者自行训练的模型和商业 API。不对训练时间、推理时间、GPU 数量、模型规模或费用进行评分。

获奖候选队伍须向组织者提交完整代码和运行环境、技术报告、外部数据与模型清单、商业 API 使用说明以及主要硬件和运行时间信息。

代码不要求公开。提交材料仅用于组织者复现，组织者应对非公开材料保密，不得用于结果复现之外的目的。

若材料不完整、结果无法合理复现或存在违规行为，组织者可在通知队伍并给予解释机会后取消资格。申诉流程、奖项、奖金及许可证将在报名开放前公布。

常见问题

检测任务是否接收置信度？

不接收。仅接收整数标签 0 和 1，检测指标为 Balanced Accuracy。

良性模型需要 embedding 吗？

若预测为良性，embedding_key 必须为 null。

误报和漏报如何影响恢复分？

漏报不产生余弦贡献；误报会增加预测后门数量和恢复分分母，因此两者都会降低恢复分。

真实目标 embedding 如何定义？

它是生成后门模型时实际使用并保存的目标特征，经 float32 转换和 L2 归一化后用于评分，不由评分器根据文本重新编码。

是否接受 .pt 文件？

不接受。目标向量统一保存在 embeddings.safetensors 中。

是否包含多目标后门？

不包含。每个后门模型只对应一个目标 embedding。

排行榜显示哪些信息？

仅显示最终总分，保留小数点后两位，不显示检测分、恢复分或逐模型结果。

获奖代码是否公开？

不要求公开，仅提交给组织者用于复现。
